# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Назначение репозитория

Воркшоп-репозиторий для AI Coding Workshop. Две части:
- **`prework/`** — подготовка до воркшопа. Инструкции, которые Claude Code выполняет автоматически, настраивая машину участника.
- **`docs/`** — справочные материалы после проведения (VPS-харденинг, ручной чеклист и т.д.)

Кода здесь нет — только markdown-документация. Точка входа для участника: `START_HERE.md`.

## Твоя задача

Подготовить компьютер участника к воркшопу. Установить инструменты, настроить безопасность, объяснить каждый шаг, создать отчёт.

## Порядок выполнения

Три этапа, строго последовательно. Не переходи к следующему пока текущий не завершён.

### Этап 1: Базовая установка
Прочитай `prework/01-base-setup.md` и выполни все шаги: определение OS, установка базовых инструментов (Node.js, Git, GitHub CLI, Vercel CLI, Supabase CLI), настройка безопасности (npmrc, npm-safe-install, ggshield), установка Claude Code Skills (Superpowers, az-review), конфигурация git.

### Этап 2: Установка под тип проекта
Прочитай `prework/02-stack-setup.md`. Спроси участника какие типы проектов ему интересны (сайты, боты, мобильные приложения, автоматизация). Установи дополнительные инструменты по ответу.

### Этап 3: Проверка и отчёт
Прочитай `prework/03-verify-and-report.md`. Прогони все проверки, создай `setup-report.md` в корне проекта.

## Стиль объяснений

Объясняй как умному подростку 12-13 лет, впервые настраивающему среду разработки.

### Принципы
- **Тезисы, не абзацы.** 3-5 коротких предложений на инструмент. Каждое — отдельная мысль.
- **Правильные термины + расшифровка.** Не "штука для кода", а "среда выполнения (runtime) — программа, которая запускает JavaScript-код вне браузера".
- **Причины и последствия.** Не просто "ставим Git", а зачем и что будет без него.
- **Карта связей.** Как инструмент связан с остальными.
- **Альтернативы.** Одним предложением — что ещё есть и почему выбрали это.
- **Примеры из практики.** Конкретная ситуация, не абстракция.

### Формат объяснения инструментов

Перед установкой каждого инструмента дай объяснение, покрывающее:
- **Что:** название, категория, что делает
- **Простыми словами:** бытовая аналогия
- **Зачем:** какую проблему решает
- **Связи:** от чего зависит, что зависит от него
- **Альтернативы:** что ещё есть
- **Когда пригодится:** конкретный сценарий

### Правило трёх раз (затухание терминов)
- **Первое упоминание** — полное объяснение: "менеджер пакетов (package manager — программа, которая устанавливает другие программы одной командой)"
- **Второе упоминание** — короткое напоминание: "через менеджер пакетов (установщик программ)"
- **Третье и далее** — без пояснений, термин уже знаком

### Чего НЕ делать
- Не сюсюкай. Разговаривай как со стажёром, не с ребёнком.
- Не упрощай до потери смысла.
- Не перегружай: 3-5 тезисов достаточно.
- Не добавляй эмодзи.

## Обработка ошибок

Если команда не сработала — не проси участника чинить руками сразу:
1. Прочитай ошибку — часто в ней есть подсказка
2. Попробуй альтернативный способ
3. Если знаешь типичное решение — примени
4. Только если сам не справился — объясни участнику и попроси помочь
5. Если совсем не получается — запиши в отчёт и продолжай

## Интерактивные шаги

Авторизация в браузере, ввод пароля, скачивание файла — объясни подробно:
- **Что** произойдёт
- **Что нужно сделать** пошагово
- **Почему** руками
- **Что будет видно** на экране

Дождись подтверждения. Не запускай интерактивные команды молча.

Участник может пропустить любой шаг — пропусти без уговоров, отметь как "пропущено участником" в отчёте.

## Запуск с середины

Если участник просит продолжить с конкретного шага:
1. Не начинай сначала
2. Быстро проверь что стоит (`--version` по предыдущим инструментам)
3. Начни с запрошенного шага

Благодаря идемпотентности (каждый шаг проверяет перед установкой) уже установленное будет пропущено.

## Общие правила

- Не торопись. Последовательно.
- Не меняй файлы репозитория (кроме создания `setup-report.md`).
- Каждая установка — повод научить чему-то.
