## 2.1 Спроси участника

Задай вопрос:

>

> 2. **Telegram-бот**

## 2.3 Telegram-бот

Нужен Python.

Проверь: `python3 --version`

Если не установлен:

**macOS:**
```bash
brew install python
```

**Linux:**
```bash
sudo apt install -y python3 python3-pip python3-venv
```

Проверь: `python3 --version` (3.10+) и `pip3 --version`

Сообщи:
> Python установлен. Библиотеки для бота (python-telegram-bot, supabase) будут установлены уже в конкретном проекте, не глобально — так правильнее.

## 2.4 Мобильное приложение (iOS / Android)

Нужны Expo CLI и EAS CLI.

```bash
npm install -g expo-cli eas-cli
```

Проверь:
```bash
expo --version
eas --version
```

**Для iOS-разработки на macOS** — проверь что Xcode установлен:
```bash
xcode-select -p
```

Если нет — скажи:
> Для тестирования iOS-приложений на симуляторе нужен Xcode. Установи его из App Store (это бесплатно, но весит ~12GB). Можно сделать это позже. Для Android — можно тестировать через Expo Go на телефоне без дополнительных установок.

Сообщи:
> Expo CLI и EAS CLI установлены. Для тестирования можно использовать приложение Expo Go на телефоне — скачай его из App Store / Google Play.

## 2.7 Деплой на VPS (Docker + hcloud)

Этот путь нужен для: Telegram-ботов на сервере, API-сервисов, self-hosted приложений, баз данных на собственном сервере.

### Docker

Проверь: `docker --version`

Если установлен — пропусти, сообщи версию.

Если не установлен:

> Docker позволяет упаковать приложение со всеми зависимостями в контейнер. Это как виртуальная машина, но легче. На сервере ваш код будет работать в Docker-контейнерах.

**macOS:**

> Для Docker на macOS нужно приложение Docker Desktop. Сейчас я проверю, установлено ли оно.

Проверь: `ls /Applications/Docker.app 2>/dev/null`

Если нет — скажи участнику:
> Docker Desktop нужно установить вручную — скачай с [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/). После установки — открой приложение Docker один раз, чтобы оно настроилось.

**Linux:**
```bash
curl -fsSL https://get.docker.com | sh
```

Проверь после установки:
```bash
docker --version
docker compose version
```

### Hetzner Cloud CLI (hcloud)

Проверь: `hcloud version`

Если установлен — пропусти, сообщи версию.

Если не установлен:

> hcloud — CLI для управления серверами на Hetzner Cloud. Hetzner — это надёжный и дешёвый хостинг для VPS (от ~4 EUR/мес). CLI позволяет создавать и управлять серверами из терминала.

**macOS:**
```bash
brew install hcloud
```

**Linux:**
```bash
brew install hcloud
```
(Или скачать бинарник: см. [github.com/hetznercloud/cli](https://github.com/hetznercloud/cli/releases))

Проверь: `hcloud version`

Сообщи:
> Docker и hcloud установлены. Для работы с Hetzner нужно будет создать аккаунт на [hetzner.com](https://www.hetzner.com) и API-токен — это делается на воркшопе или после. VPS hardening runbook лежит в `workshop/prework/vps-hardening-runbook.md`.
