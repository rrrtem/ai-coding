# Этап 2: Установка под тип проекта

> Инструкции для Claude Code.
> Принцип идемпотентности: перед установкой каждого инструмента — проверь, установлен ли он уже. Если да — пропусти. Подробности в 01-base-setup.md.

## 2.1 Спроси участника

Задай вопрос:

> Что ты хочешь сделать на воркшопе (и после)? Можно выбрать несколько вариантов:
>
> 1. **Сайт или лендинг** — страница для продукта, портфолио, бизнеса
> 2. **Веб-приложение с личным кабинетом** — SaaS, дашборд, CRM, внутренний инструмент с логином и данными пользователей
> 3. **Telegram-бот** — бот который отвечает клиентам, обрабатывает заказы, присылает уведомления
> 4. **Мобильное приложение** — приложение для iPhone или Android
> 5. **Автоматизация** — парсинг сайтов, обработка таблиц/данных, интеграция сервисов, рассылки
> 6. **Что-то другое** — опиши в 1-2 предложениях, я разберусь
>
> Если не уверен — выбери 1 или 2. Это покрывает большинство случаев.

Дождись ответа. Участник может выбрать несколько вариантов.

## 2.2 Определи что нужно установить

На основе ответа участника определи набор инструментов по таблице ниже. Если выбрано несколько вариантов — объедини все нужные инструменты.

| Что хочет участник | Что установить дополнительно | Почему |
|---|---|---|
| Сайт / лендинг | Ничего | Всё из базовой установки: Vercel опубликует сайт, Supabase даст формы/данные если нужно |
| Веб-приложение с ЛК | Ничего | Всё из базовой установки: Supabase Auth для логина, Vercel для хостинга |
| Telegram-бот | Python, Docker, hcloud | Бот должен работать 24/7 → нужен сервер. Docker упакует бота, hcloud создаст сервер на Hetzner (~4 EUR/мес) |
| Мобильное приложение | expo-cli, eas-cli | Expo собирает приложения из JavaScript в настоящие iOS/Android приложения |
| Автоматизация | Python | Python — стандарт для скриптов, парсинга и обработки данных |
| Что-то другое | CC решает сам | Проанализируй описание участника и определи нужные инструменты из списка выше |

### Перед установкой — объясни и подтверди

Покажи участнику что ты собираешься установить и зачем. Пример:

> Ты выбрал **Telegram-бота** и **сайт**.
>
> Для сайта всё уже установлено (Vercel + Supabase из базовой установки).
>
> Для бота мне нужно доставить:
> - **Python** — язык на котором пишется бот
> - **Docker** — упаковка бота для сервера (как коробка со всем необходимым)
> - **hcloud** — управление сервером на Hetzner (дешёвый хостинг, ~4 EUR/мес)
>
> Бот должен работать круглосуточно, поэтому ему нужен свой сервер — в отличие от сайта, который живёт на Vercel бесплатно.
>
> Ставлю?

Дождись подтверждения. Затем выполни установку нужных компонентов из секций ниже.

## 2.3 Python

Нужен для: Telegram-ботов, автоматизаций, бэкенд-сервисов на FastAPI.

Проверь: `python3 --version`

Если установлен (3.10+) — пропусти, сообщи версию.

Если не установлен:

> Python — универсальный язык для ботов, скриптов и обработки данных. Одна установка — и он доступен для всех проектов.

**macOS:**
```bash
brew install python
```

**Linux:**
```bash
sudo apt install -y python3 python3-pip python3-venv
```

Проверь: `python3 --version` (3.10+) и `pip3 --version`

## 2.4 Docker

Нужен для: Telegram-ботов (деплой на сервер), API-сервисов, баз данных на своём сервере.

Проверь: `docker --version`

Если установлен — пропусти, сообщи версию.

Если не установлен:

> Docker упаковывает приложение со всеми зависимостями в контейнер — как коробку которую можно поставить на любой сервер и она будет работать одинаково. Без Docker деплой на сервер превращается в ручную настройку.

**macOS:**

Проверь: `ls /Applications/Docker.app 2>/dev/null`

Если нет — скажи участнику:
> Docker Desktop нужно скачать и установить вручную — [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/). После скачивания — перетащи в Applications и открой один раз, чтобы он настроился.

**Linux:**
```bash
curl -fsSL https://get.docker.com | sh
```

Проверь после установки:
```bash
docker --version
docker compose version
```

## 2.5 hcloud (управление сервером)

Нужен для: Telegram-ботов (деплой на сервер), API-сервисов, любых проектов которые должны работать 24/7.

Проверь: `hcloud version`

Если установлен — пропусти, сообщи версию.

Если не установлен:

> hcloud — управление серверами на Hetzner Cloud из терминала. Hetzner — надёжный хостинг из Германии, сервер стоит от ~4 EUR/мес. Аккаунт на Hetzner создадим позже, сейчас ставим только CLI.

**macOS:**
```bash
brew install hcloud
```

**Linux:**
```bash
brew install hcloud
```
(Или скачать бинарник: [github.com/hetznercloud/cli](https://github.com/hetznercloud/cli/releases))

Проверь: `hcloud version`

Сообщи:
> hcloud установлен. Для работы с Hetzner нужно будет создать аккаунт — это делается на воркшопе или после. Инструкция по настройке и безопасности сервера лежит в `workshop/prework/vps-hardening-runbook.md`.

## 2.6 Expo (мобильные приложения)

Нужен для: iOS и Android приложений.

Проверь: `expo --version` и `eas --version`

Если установлены — пропусти, сообщи версии.

Если не установлены:

> Expo — платформа которая позволяет писать мобильные приложения на JavaScript. Один код → приложения для iPhone и Android. EAS — сервис сборки (собирает готовые .apk / .ipa файлы).

```bash
npm install -g expo-cli eas-cli
```

**Для iOS на macOS** — проверь Xcode:
```bash
xcode-select -p
```

Если нет:
> Для тестирования на iPhone-симуляторе нужен Xcode (бесплатно в App Store, ~12GB). Можно поставить позже. Для быстрого тестирования на реальном телефоне — скачай приложение **Expo Go** из App Store / Google Play.
