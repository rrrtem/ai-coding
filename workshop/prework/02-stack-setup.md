## 2.1 Спроси участника

Задай вопрос:

> Что ты хочешь сделать на воркшопе (и после)? Можно выбрать несколько вариантов:
>
> 1. **Сайт или лендинг** — страница для продукта, портфолио, бизнеса
> 2. **Веб-приложение с личным кабинетом** — сервис с логином и данными пользователей (например: панель управления заказами, система учёта клиентов, внутренний инструмент для команды)
> 3. **Telegram-бот** — бот который отвечает клиентам, обрабатывает заказы, присылает уведомления
> 4. **Мобильное приложение** — приложение для iPhone или Android
> 5. **Автоматизация** — парсинг сайтов, обработка таблиц/данных, интеграция сервисов, рассылки
> 6. **Что-то другое** — опиши в 1-2 предложениях, я разберусь
>
> Если не уверен — выбери 1 или 2. Это покрывает большинство случаев.

Дождись ответа. Участник может выбрать несколько вариантов.

| Что хочет участник | Что установить дополнительно | Почему |
|---|---|---|
| Сайт / лендинг | Ничего | Всё из базовой установки: Vercel опубликует сайт, Supabase даст формы/данные если нужно |

## 2.3 Python

Нужен для: Telegram-ботов, автоматизаций, бэкенд-сервисов на FastAPI.

Проверь: `python3 --version`

Если установлен (3.10+) — пропусти, сообщи версию.

Если не установлен — объясни:

> **Что:** Python — интерпретируемый язык программирования общего назначения. "Интерпретируемый" значит код выполняется строчка за строчкой, без предварительной компиляции — написал, запустил, увидел результат.
> **Простыми словами:** Если JavaScript — это швейцарский нож для веба, то Python — швейцарский нож для всего остального: ботов, скриптов, анализа данных. Код на Python читается почти как английский текст — проще начать.
> **Зачем:** Python — стандарт для ботов, скриптов (небольших программ-автоматизаций), обработки данных и машинного обучения. У него огромная экосистема библиотек (готового кода для конкретных задач): `python-telegram-bot` для Telegram-ботов, `pandas` для работы с таблицами, `requests` для отправки запросов к другим сервисам. Без Python — пришлось бы писать ботов на JavaScript, где библиотеки для этого менее зрелые.
> **Связи:** Python (язык) → pip (менеджер пакетов, аналог npm) → библиотеки проекта. Docker упаковывает Python-приложение для деплоя на сервер. Python независим от Node.js — это параллельный стек.
> **Альтернативы:** Go, Rust — быстрее, но сложнее в освоении. JavaScript (Node.js) — можно, но для ботов и скриптов Python удобнее: меньше кода, больше готовых библиотек.
> **Когда пригодится:** Ты хочешь написать Telegram-бота, который отвечает на сообщения клиентов. На Python это 20 строк кода с библиотекой `python-telegram-bot`. На JavaScript — в 2-3 раза больше, и библиотеки менее зрелые.

**macOS:**
```bash
brew install python
```

**Linux:**
```bash
sudo apt install -y python3 python3-pip python3-venv
```

Проверь: `python3 --version` (3.10+) и `pip3 --version`

## 2.4 Docker

Нужен для: Telegram-ботов (деплой на сервер), API-сервисов, баз данных на своём сервере.

Проверь: `docker --version`

Если установлен — пропусти, сообщи версию.

Если не установлен — объясни:

> **Что:** Docker — платформа контейнеризации. Контейнер (container) — это изолированная среда, в которой приложение запускается вместе со всеми зависимостями: языком, библиотеками, настройками.
> **Простыми словами:** Как коробка для переезда, в которую ты складываешь не только вещи, но и саму комнату — мебель, обои, розетки. Ставишь коробку в любое место — и внутри всё выглядит одинаково. Docker делает то же самое с программами.
> **Зачем:** Без Docker деплой на сервер — ручная работа: ставишь Python нужной версии, ставишь библиотеки, настраиваешь окружение, и молишься что всё заработает так же, как на твоём компьютере. Docker упаковывает всё в один файл (образ), который работает одинаково везде.
> **Связи:** Ты пишешь код → Dockerfile описывает окружение → Docker собирает образ (image) → образ запускается на сервере (Hetzner через hcloud). Без Docker деплой возможен, но в 5 раз сложнее и ненадёжнее.
> **Альтернативы:** Podman — аналог Docker без демона (фоновой службы). Но Docker — стандарт индустрии, и все туториалы написаны под него.
> **Когда пригодится:** Ты написал Telegram-бота. Он работает на твоём компьютере. Теперь надо запустить его на сервере, чтобы он работал 24/7. Без Docker: заходишь на сервер, ставишь Python, копируешь файлы, ставишь библиотеки, запускаешь — и если что-то отличается от твоего компьютера, бот падает. С Docker: `docker compose up` — и бот работает.

**macOS:**

Проверь: `ls /Applications/Docker.app 2>/dev/null`

Если нет — **это единственный инструмент, который нельзя поставить из терминала на macOS.** Предупреди участника и объясни пошагово:

> Сейчас нужно будет сделать кое-что руками — Docker Desktop на macOS нельзя установить командой, только через браузер. Вот что нужно сделать:
>
> 1. Открой браузер и перейди на [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/)
> 2. Нажми кнопку "Download for Mac" (выбери Apple Chip если у тебя M1/M2/M3/M4, или Intel Chip если старый Mac)
> 3. Скачается файл `Docker.dmg` — открой его
> 4. Перетащи иконку Docker в папку Applications
> 5. Открой Docker из Applications один раз — он настроится (это займёт минуту)
> 6. Когда в верхней панели появится иконка кита — Docker готов
>
> Скажи мне когда закончишь, и я проверю что всё работает.

Дождись подтверждения участника.

**Linux:**
```bash
curl -fsSL https://get.docker.com | sh
```

Проверь после установки:
```bash
docker --version
docker compose version
```

## 2.5 hcloud (управление сервером)

Нужен для: Telegram-ботов (деплой на сервер), API-сервисов, любых проектов которые должны работать 24/7.

Проверь: `hcloud version`

Если установлен — пропусти, сообщи версию.

Если не установлен — объясни:

> **Что:** hcloud — CLI для управления серверами на Hetzner Cloud. Hetzner — немецкий облачный провайдер (cloud provider), который сдаёт в аренду виртуальные серверы (VPS).
> **Простыми словами:** Как аренда квартиры, но для программ. Тебе не нужно покупать свой компьютер и держать его включённым 24/7 — ты арендуешь кусочек мощного сервера в дата-центре за ~4 EUR/мес. hcloud — это приложение для управления этой "квартирой".
> **Зачем:** Telegram-бот или API должен работать 24/7. Для этого нужен сервер — компьютер, который всегда включён и подключён к интернету. Hetzner даёт такой сервер за ~4 EUR/мес. hcloud позволяет создавать, настраивать и удалять серверы одной командой из терминала.
> **Связи:** Ты пишешь бота → Docker упаковывает → hcloud создаёт сервер → ты деплоишь Docker-образ на сервер. hcloud — последнее звено цепочки: без него негде запустить бота.
> **Альтернативы:** AWS, Google Cloud, DigitalOcean — другие облачные провайдеры. Hetzner выбран за цену (в 2-3 раза дешевле AWS), простоту, и дата-центры в Европе (меньше задержка для российских/европейских пользователей).
> **Когда пригодится:** Бот готов и работает на твоём компьютере. Ты закрываешь ноутбук — бот перестаёт отвечать. Команда `hcloud server create --type cx22 --image ubuntu-24.04 --name my-bot` — и через 30 секунд у тебя сервер, который работает круглосуточно. Аккаунт на Hetzner создадим позже, сейчас ставим только CLI.

**macOS:**
```bash
brew install hcloud
```

**Linux (без Homebrew):**
```bash
# Скачай последний релиз напрямую:
curl -fsSL https://github.com/hetznercloud/cli/releases/latest/download/hcloud-linux-amd64.tar.gz | tar -xz -C /usr/local/bin hcloud
```
(Или через Homebrew если он установлен: `brew install hcloud`)

Проверь: `hcloud version`

Сообщи:
> hcloud установлен. Для работы с Hetzner нужно будет создать аккаунт — это делается на воркшопе или после. Инструкция по настройке и безопасности сервера лежит в `workshop/prework/vps-hardening-runbook.md`.

## 2.6 Expo (мобильные приложения)

Нужен для: iOS и Android приложений.

Проверь: `npx expo --version` и `eas --version`

Если `eas` установлен — пропусти, сообщи версию.

Если не установлены — объясни:

> **Что:** Expo — платформа для разработки мобильных приложений на JavaScript. EAS (Expo Application Services) — облачный сервис сборки, который превращает код в готовые .apk (Android) и .ipa (iOS) файлы.
> **Простыми словами:** Обычно для iPhone нужен один язык (Swift), для Android — другой (Kotlin). Это как писать одну и ту же книгу на двух языках. Expo — это переводчик: ты пишешь один раз на JavaScript, а он делает версию и для iPhone, и для Android.
> **Зачем:** Без Expo — два языка, две кодовых базы, двойная работа. С Expo — один код, две платформы. Плюс не нужно разбираться в сложной настройке Xcode (Apple) и Android Studio (Google).
> **Связи:** Node.js (запускает Expo) → Expo CLI (создаёт проект, запускает сервер для разработки) → EAS CLI (собирает финальное приложение в облаке). Expo использует React Native под капотом — фреймворк (готовый каркас) от Facebook для мобильных приложений.
> **Альтернативы:** React Native CLI (без Expo) — больше контроля, но сложнее настройка. Flutter (от Google, свой язык Dart) — тоже хорош, но требует изучения нового языка. Expo выбран потому что проще всего начать: один `npx create-expo-app` — и через минуту ты видишь приложение на телефоне.
> **Когда пригодится:** Ты хочешь сделать мобильное приложение. Скачиваешь Expo Go на телефон, запускаешь `npx expo start`, сканируешь QR-код — и видишь своё приложение прямо на телефоне в реальном времени. Меняешь код — приложение обновляется мгновенно.

```bash
npm install -g eas-cli
```

> `expo-cli` устарел (deprecated). Вместо глобальной установки используй `npx expo` (или `npx create-expo-app`) — Expo автоматически скачает нужную версию. Глобально нужен только `eas-cli` для сборки приложений.

**Для iOS на macOS** — проверь Xcode:
```bash
xcode-select -p
```

Если нет:
> Для тестирования на iPhone-симуляторе нужен Xcode (бесплатно в App Store, ~12GB). Можно поставить позже. Для быстрого тестирования на реальном телефоне — скачай приложение **Expo Go** из App Store / Google Play.
